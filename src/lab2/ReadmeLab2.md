# Задание 1
## Точка координат
### Текст задачи
Создайте сущность Точка, расположенную на двумерной плоскости, которая описывается: 
- Координата Х: число 
- Координата Y: число 
- Может возвращать текстовое представление вида “{X;Y}” 
Необходимо создать три точки с разными координатами и вывести на экран их текстовое 
представление.
### Алгоритм решения
1. Создать класс `Point` с соответствующими полями `private double x` и `private double y`, свойствами `get`, `set`, конструктором по умолчанию `public Point()`, конструктором с двумя аргументами `public Point(double x, double y)` и переопределённым методом `toString()`.
2. В основном классе трижды создать объекты `Point` с разными координатами и вывести их текстовое представление.
3. Каждый раз создаётся новая точка, так как результаты не требуется сохранять.
### Тесты
input: 
3 0
2 10
-3 4.6  
output: 
{3.0;0.0}
{2.0;10.0}
{-3.0;4.6}


# Задание 2
## Прямая
### Текст задачи
Создайте сущность Линия, расположенную на двумерной плоскости, которая описывается: 
- Координата начала: Точка 
- Координата конца: Точка 
- Может возвращать текстовое представление вида “Линия от {X1;Y1} до {X2;Y2}” 
Для указания координат нужно использовать сущность Точка, разработанную в задании 1.1. Создайте 
линии со следующими характеристиками: 
1. Линия 1 с началом в т. {1;3} и концом в т.{23;8}. 
2. Линия 2, горизонтальная, на высоте 10, от точки 5 до точки 25. 
3. Линия 3, которая начинается всегда там же, где начинается линия 1, и заканчивается всегда там 
же, где заканчивается линия 2. Таким образом, если положение первой или второй линии 
меняется, то меняется и третья линия. 
4. После создания всех трех объектов измените координаты первой и второй линий, причем 
сделайте это таким образом, чтобы положение третьей линии соответствовало требованиям 
пункта 3. 
5. Измените координаты первой линии так, чтобы при этом не изменились, координаты третьей 
линии. 
### Алгоритм решения
1. Создать класс `Line` с полями `Point startPoint` и `Point endPoint` типа `Point`, свойствами get и set, конструктором `public Line(Point startPoint, Point endPoint)` и переопределённым методом `toString()`.
2. В основном классе создать точки и линии, передавая объекты Point в конструкторы.
3. Чтобы линия 3 зависела от линии 1 и 2, её точки должны ссылаться на те же объекты, что и точки этих линий.
4. Чтобы изменение первой линии не влияло на третью, необходимо присвоить новой первой точке линии 1 новый объект Point, а не изменить координаты старого.
### Тесты
input:  
p1(1, 3), p2(23, 8)  
l1(p1, p2)  
p1(5, 10), p2(25, 10)  
l2(p3, p4)  
l3(p1, p4)  
output:  
1 Линия от {1.0;3.0} до {23.0;8.0}  
2 Линия от {5.0;10.0} до {25.0;10.0}  
3 Линия от {1.0;3.0} до {25.0;10.0}    
Изменяем координаты 1 и 2 динии:  
input:  
p1.x = 9, p1.y = 0  
p4.x = -24, p4.y = -3   
output:   
Изменённые координаты:  
1 Линия от {9.0;0.0} до {25.0;-3.0}  
2 Линия от {45.0;2.0} до {-24.0;-3.0}  
3 Линия от {9.0;0.0} до {-24.0;-3.0}   
input:   
p1 = (new Point(10, 20))  
p2 = (new Point(1, 43))    
output:  
Изменённые координаты первой линии: 
1 Линия от {10.0;20.0} до {1.0;43.0}  
2 Линия от {45.0;2.0} до {-24.0;-3.0}  
3 Линия от {9.0;0.0} до {-24.0;-3.0}  


# Задание 3
## Студент
### Текст задачи
Создайте сущность Студент, которая описывается: 
- Имя: строка 
- Оценки: массив целых чисел. 
- Может возвращать текстовое представление вида “Имя: [оценка1, оценка2,…,оценкаN]” 
Необходимо выполнить следующие задачи: 
1. Создать студента Васю с оценками: 3,4,5.  
2. Создать студента Петю и скопировать оценки Васи, присвоив содержимое поля с 
оценками Васи полю с оценками Пети.  
3. Заменить первую оценку Пети на число 5. Вывести на экран строковое представление 
Васи и Пети. Объяснить результат 
4. Создать студента Андрея и скопировать ему оценки Васи так, чтобы изменение оценок 
Васи не влияло на Андрея. 
### Алгоритм решения
1. Создать класс `Student` с полями `private String name` и `private int[] grades`, свойствами get, set, конструкторами `public Student(String name)`, `public Student(String name, int[] grades)` и переопределённым методом `toString()`.
2. При копировании массива оценок у Васи в поле Пети получили ссылку на один массив, поэтому при изменении первой оценки у Пети также меняются оценки у Васи.
3. Чтобы изенение оценок у Андрея не меняло оценки у Васи, делаем копию массива.
### Тесты
input:  
"Вася", new int[]{1, 2, 3}  
"Петя", petya.setGrades(vasya.getGrades());  
        petya.getGrades()[0] = 5;  
output:  
Вася: [1, 2, 3]  
Изменили первую оценку у Пети:  
Вася: [5, 2, 3]  
Петя: [5, 2, 3]   
input:  
"Андрей", int[] arr = vasya.getGrades();  
          andrey.setGrades(Arrays.copyOf(arr, arr.length));   
output:  
Скопировали оценки Васи:  
Андрей: [5, 2, 3]  
Изменили оценки Васи:  
Вася: [5, 5, 3]  
Андрей: [5, 2, 3]

# Задание 5
## Студент отличник
### Текст задачи
Измените сущность Студент из задачи 4.7. Новые требования включают: 
 Студент может возвращать значение своей средней оценки (вещественное число). Если 
оценок нет, то среднее – 0. 
 Студент может возвращать информацию о том, является ли он отличником или нет. 
Студент является отличником если у него есть хотя бы одна оценка и все оценки только 
отличные. 
Необходимо выполнить следующие задачи: 
1. Создайте студента Васю с оценками 3,4,5,4 
2. Создайте студента Петю с оценками 5,5,5,5 
3. Выведите для обоих студентов информацию о среднем балле и являются ли они 
отличниками 
### Алгоритм решения
1. Добавить в класс `Student` метод `getAverage()` - возвращает среднюю оценку (считает среднее значение элементов массива).
2. Добавить метод `isTop()`, который проверяет что количество оценок юольше или равно 1 и средняя оценка = 5.
### Тесты
output:  
Вася: средняя оценка: 4.0  
Вася не отличник  
Петя: средняя оценка: 5.0  
Петя отличник
